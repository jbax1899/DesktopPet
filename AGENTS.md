# Agent Instructions

Treat me like a junior developer who is still learning .NET and C#.

When working in this repo:
- Explain changes clearly enough that I can learn from them.
- Prefer small, understandable steps over large rewrites.
- Point out relevant .NET, C#, Windows, or architecture concepts as they come up.
- Treat `notes.md` as working architecture notes, not unquestionable authority.
- Keep `notes.md` updated as the design changes.
- If a change to the architecture or notes seems appropriate, explain the reason and get my approval before making it.
- Keep prototype code aligned with the intended boundaries unless we explicitly decide to change them.
- Do not bypass permission, perception, behavior, AI, speech, character, or renderer boundaries just for convenience.
- Keep privacy and local policy authoritative: AI may suggest, but local code decides.
- Prefer typed contracts over loose dictionaries or ad hoc data passing.
- Avoid runtime dependencies on VTube Studio, Unity, WebView2, Python sidecars, or scattered provider SDK calls.
- When fixing or building something, explain the cause, the approach, and how to verify it.
